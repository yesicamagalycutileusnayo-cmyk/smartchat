📖 Capítulo 1

Introducción

👤 Equipo SmartChat 📅 2025 ⏱️ ~10 min lectura

1.1 Contexto y Problemática

En la actualidad, los problemas de salud mental representan una de las principales preocupaciones a nivel mundial. Según la Organización Mundial de la Salud (OMS), más del 30% de la población estudiantil universitaria experimenta niveles significativos de estrés, ansiedad y depresión durante su formación académica.

La falta de acceso a apoyo psicológico oportuno, combinada con el estigma social asociado a la salud mental, genera barreras que impiden a los jóvenes buscar ayuda cuando más la necesitan. Los servicios de atención psicológica presentan limitaciones de horario, costos elevados y disponibilidad reducida.

💡 SmartChat surge como respuesta tecnológica a esta problemática, ofreciendo un primer punto de contacto accesible, disponible las 24 horas y sin barreras económicas.

1.2 Objetivo General

Desarrollar un sistema web interactivo de apoyo emocional basado en simulación de inteligencia artificial, que proporcione respuestas empáticas y orientación básica a usuarios que experimenten estados emocionales negativos como tristeza, ansiedad, estrés o soledad.

1.3 Objetivos Específicos

  • Diseñar e implementar una interfaz web moderna tipo SaaS con experiencia de usuario premium.
  • Desarrollar un motor de respuestas basado en detección de palabras clave emocionales.
  • Implementar un sistema de autenticación simulada con gestión de sesiones mediante localStorage.
  • Crear un dashboard analítico con visualización de estadísticas de uso.
  • Documentar el proceso de desarrollo siguiendo estándares de ingeniería de software.
  • Implementar diseño responsive compatible con dispositivos móviles y de escritorio.

1.4 Justificación

El proyecto SmartChat se justifica desde tres dimensiones fundamentales:

  • Social: Democratiza el acceso a herramientas de apoyo emocional, reduciendo barreras económicas y geográficas.
  • Tecnológica: Aplica técnicas modernas de desarrollo web frontend para crear experiencias interactivas de alto valor.
  • Académica: Integra conocimientos de ingeniería de software, diseño UX/UI, psicología cognitivo-conductual y gestión de proyectos.

1.5 Alcance del Sistema

✅ SmartChat es un sistema 100% frontend con simulación de respuestas. No incluye backend real, base de datos externa ni servicios de IA. Las conversaciones son locales y no se almacenan en ningún servidor.

El sistema incluye: landing page informativa, sistema de login simulado, dashboard de control, chatbot emocional interactivo y documentación académica estructurada.

2.2 Requerimientos No Funcionales

  • Rendimiento: El tiempo de carga inicial debe ser menor a 3 segundos en conexiones estándar.
  • Usabilidad: La interfaz debe ser intuitiva y no requerir capacitación previa.
  • Compatibilidad: Compatible con Chrome, Firefox, Edge y Safari en sus versiones actuales.
  • Accesibilidad: Cumplir con estándares WCAG 2.1 nivel AA (etiquetas ARIA, contraste de colores).
  • Responsive: Adaptable a pantallas desde 320px hasta 2560px de ancho.
  • Seguridad: No transmitir datos sensibles a servidores externos.

2.4 Casos de Uso Principales

  1. CU-01 — Acceder al Sistema: El usuario ingresa sus credenciales. El sistema valida y redirige al dashboard.
  2. CU-02 — Iniciar Conversación: El usuario escribe un mensaje. El chatbot detecta el estado emocional y responde.
  3. CU-03 — Cambiar Tema: El usuario alterna entre modo claro y oscuro. El sistema persiste la preferencia.
  4. CU-04 — Consultar Documentación: El usuario navega por los capítulos del informe académico.
  5. CU-05 — Cerrar Sesión: El usuario termina la sesión. El sistema limpia los datos de localStorage.

Diseño del Sistema

📅 2025⏱️ ~15 min lectura

3.1 Arquitectura del Sistema

SmartChat implementa una arquitectura 100% frontend sin servidor, también conocida como arquitectura Jamstack simplificada. Todos los módulos del sistema se ejecutan en el navegador del cliente.

SmartChat/ ├── index.html → Landing Page (punto de entrada) ├── login.html → Autenticación simulada ├── pages/ │ ├── dashboard.html → Panel de control │ ├── chatbot.html → Interfaz del chatbot │ ├── documentacion.html → Documentación académica │ └── contacto.html → Formulario de contacto ├── css/ │ ├── main.css → Sistema de diseño global │ ├── landing.css → Estilos de la landing │ ├── login.css → Estilos del login │ ├── dashboard.css → Estilos del dashboard │ ├── chatbot.css → Estilos del chatbot │ └── docs.css → Estilos de documentación └── js/ ├── theme.js → Gestión de temas ├── auth.js → Autenticación y sesión ├── landing.js → Interacciones de la landing ├── dashboard.js → Lógica del dashboard └── chatbot.js → Motor del chatbot

3.2 Sistema de Diseño

Se implementó un design system coherente con variables CSS custom properties que garantizan consistencia visual en todo el sistema:

  • Tipografía: Inter (Google Fonts) — pesos 300 a 900.
  • Paleta primaria: Azul (#2563EB) y Cian (#06B6D4).
  • Espaciado: Sistema de 8px base (8, 16, 24, 32, 48, 64, 96px).
  • Bordes: Sistema de radios (6, 12, 18, 24px, full).
  • Sombras: 5 niveles de profundidad (xs, sm, md, lg, xl).
  • Animaciones: 0.25s cubic-bezier para transiciones estándar.

3.3 Diseño de Interfaz de Usuario

La interfaz sigue principios de diseño moderno tipo SaaS, inspirada en plataformas como ChatGPT, Notion y Linear:

🎨 Se aplicaron principios de glassmorphism, micro-animaciones y jerarquía visual clara para crear una experiencia premium que inspire confianza en el usuario.
  • Navbar con efecto blur glassmorphism al hacer scroll.
  • Sidebar fijo con navegación contextual activa.
  • Cards con sombras suaves y efectos hover.
  • Chat UI tipo WhatsApp con burbujas diferenciadas.
  • Animaciones de fade-in al entrar en viewport (Intersection Observer).

4.2 Motor del Chatbot Emocional

El chatbot implementa un sistema de detección basado en base de conocimiento (KB). Cada regla contiene un conjunto de palabras clave y un arreglo de respuestas posibles:

const KB = [ { keys: ['triste', 'tristeza', 'llorar', 'deprimido'], responses: [ 'Entiendo que estás pasando por un momento difícil...', 'Está bien sentirse triste a veces...', ] }, // ... más reglas emocionales ]; function getResponse(message) { const lower = normalize(message.toLowerCase()); for (const rule of KB) { if (rule.keys.some(k => lower.includes(k))) { return randomFrom(rule.responses); } } return randomFrom(DEFAULT_RESPONSES); }
📝 El sistema normaliza el texto eliminando tildes y convirtiendo a minúsculas antes de comparar, lo que mejora significativamente la tasa de detección.

4.3 Sistema de Autenticación

La autenticación simula el flujo de un sistema real usando el módulo AUTH:

  • Credenciales predefinidas: admin@demo.com / 1234
  • Sesión almacenada en localStorage como JSON serializado.
  • Protección de rutas mediante AUTH.requireAuth() en cada página protegida.
  • Logout elimina completamente los datos de sesión.

🏁 Capítulo 7

Conclusiones y Recomendaciones

📅 2025⏱️ ~8 min lectura

7.1 Conclusiones

  • SmartChat demuestra que es posible crear una experiencia de apoyo emocional accesible utilizando únicamente tecnologías frontend estándar, sin requerir infraestructura de backend.
  • El sistema de detección por palabras clave, aunque básico, resulta efectivo para identificar los principales estados emocionales y proporcionar respuestas contextualizadas.
  • La implementación de un design system coherente desde el inicio del proyecto garantizó consistencia visual y redujo significativamente el tiempo de desarrollo.
  • El uso de CSS Custom Properties para la gestión de temas permitió implementar el modo oscuro con mínimo código adicional y máxima eficiencia.
  • La gestión de sesiones con localStorage, aunque limitada, es suficiente para simular un flujo de autenticación completo en un entorno académico.

7.2 Lecciones Aprendidas

  • La planificación detallada del sistema de diseño antes de escribir código HTML reduce significativamente el retrabajo posterior.
  • Las APIs nativas del navegador (Intersection Observer, localStorage, CSS Variables) son suficientemente poderosas para construir aplicaciones modernas sin frameworks.
  • La accesibilidad web (atributos ARIA, semántica HTML5) debe considerarse desde el diseño, no como un ajuste posterior.
  • El diseño mobile-first simplifica el desarrollo responsive y mejora la experiencia en todos los dispositivos.

7.3 Recomendaciones para Versiones Futuras

  1. Backend real: Implementar un servidor Node.js/Python con API REST para persistencia de conversaciones.
  2. IA avanzada: Integrar la API de OpenAI o Google Gemini para respuestas más contextuales y personalizadas.
  3. Base de datos: Usar Firebase o Supabase para almacenamiento en tiempo real de conversaciones y usuarios.
  4. Análisis de sentimientos: Implementar NLP (Natural Language Processing) para mejor comprensión emocional.
  5. Notificaciones: Sistema de recordatorios y check-ins periódicos via PWA (Progressive Web App).
  6. Panel de expertos: Módulo para que psicólogos monitoren conversaciones y puedan intervenir cuando sea necesario.
🎓 SmartChat v1.0 cumple satisfactoriamente con todos los objetivos planteados al inicio del proyecto académico. El sistema representa una base sólida para el desarrollo de soluciones de salud mental digital accesibles.
